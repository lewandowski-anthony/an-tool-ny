# k6 Cheatsheet

> A practical guide to [k6](https://k6.io/), a modern load testing tool that feels natural for developers. Tests are written in **JavaScript (ES6)** and run on a fast Go engine. For general concepts, see `performance-testing.md`.

---

## Install & Run

```bash
# Install
brew install k6                      # macOS
winget install k6 --source winget    # Windows
sudo apt-get install k6              # Linux (after adding the repo)

# Run
k6 run script.js
k6 run --vus 50 --duration 30s script.js      # override VUs/duration
k6 run -e BASE_URL=https://api.example.com script.js   # env var
k6 run --out json=result.json script.js       # export raw results
```

## Minimal Test

```js
import http from 'k6/http';
import { check, sleep } from 'k6';

export const options = {
  vus: 10,          // virtual users
  duration: '30s',  // total run time
};

export default function () {
  const res = http.get('https://test.k6.io');
  check(res, {
    'status is 200': (r) => r.status === 200,
    'body not empty': (r) => r.body.length > 0,
  });
  sleep(1);         // think-time
}
```

* `default function` = the code each VU runs in a loop.
* `check()` = soft assertions (don't stop the test, but recorded).
* `sleep()` = simulate user think-time.

## Load Profiles (stages)

```js
export const options = {
  stages: [
    { duration: '1m', target: 100 },   // ramp up to 100 VUs
    { duration: '3m', target: 100 },   // stay at 100 (load)
    { duration: '1m', target: 200 },   // spike
    { duration: '2m', target: 0 },     // ramp down
  ],
};
```

### Executors (advanced scenarios)
```js
export const options = {
  scenarios: {
    constant_rps: {
      executor: 'constant-arrival-rate',  // fix throughput, not VUs
      rate: 500, timeUnit: '1s',
      duration: '5m',
      preAllocatedVUs: 100, maxVUs: 500,
    },
  },
};
```
Executors: `shared-iterations`, `per-vu-iterations`, `constant-vus`, `ramping-vus`, `constant-arrival-rate`, `ramping-arrival-rate`.

## Thresholds (pass/fail for CI)

```js
export const options = {
  thresholds: {
    http_req_duration: ['p(95)<300', 'p(99)<800'],  // latency SLOs
    http_req_failed:   ['rate<0.01'],               // <1% errors
    checks:            ['rate>0.99'],               // >99% checks pass
  },
};
```
> **Tip:** If a threshold is breached, `k6` exits non-zero and fails your CI pipeline.

## Ecosystem

* **k6 Cloud / Grafana Cloud k6** — managed runs and dashboards.
* **xk6** — build custom binaries with extensions (SQL, Kafka, gRPC, browser).
* **k6 browser** — real-browser (Chromium) testing for frontend performance.
* Outputs: JSON, CSV, InfluxDB + Grafana, Prometheus, Datadog.

---

## Notes & Gotchas

* k6 has **no DOM/browser by default** — it's protocol-level (HTTP/WS/gRPC). Use k6 browser module for real browsers.
* Scripts run in a k6 JS runtime (goja), **not Node** — no `require`, limited npm compat (bundle with webpack/rollup if needed).
* `check()` failures don't fail the test — use **thresholds** for CI gating.
* Each VU runs the `default` function repeatedly; keep per-iteration state minimal.
* Prefer `SharedArray` for large test data to avoid copying per VU.
